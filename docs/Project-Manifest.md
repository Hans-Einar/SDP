# Project Manifest

`SDP/SDP-project.manifest.yaml` is project-owned and conforms to
`Toolkit/schemas/SDP-project-manifest.schema.json`.

It records project release state, next target, active Sprint or Refactor,
Iteration, Slice, Fix/revision, latest confirmed release identity, migration
warnings and project capabilities.

Installed Toolkit facts are not duplicated manually. The project manifest points
to `Framework/installed-toolkit.manifest.yaml`, which the installer generates
with Toolkit, Framework, AGENTS contract, installer and skill versions plus the
installation timestamp. This separation prevents stale copied values.

Dynamic Git SHA, dirty state and build timestamp are generated by
`Toolkit/scripts/New-SDPBuildIdentity.py`; they are not persisted as editable
project facts.
