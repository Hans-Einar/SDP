# Project Manifest

`SDP/SDP-project.manifest.yaml` is project-owned and conforms to
`Toolkit/schemas/SDP-project-manifest.schema.json` from a compatible Toolkit.
Its neutral source is an explicit missing-only entry under
`Toolkit/project-templates/`; no conforming installer replaces it, including
under force.

It records the consuming project's release state, next target, active Sprint or
Refactor, Iteration, Slice, Fix/revision, latest confirmed release identity,
migration warnings and project capabilities. A new template contains neutral
project values only. It never inherits the SDP Toolkit repository's
`REL-0.2.0`, active work coordinates or publication claims.

Installed Toolkit facts are not duplicated manually. The project manifest
points, relative to the project `SDP/` directory, to
`Framework/installed-toolkit.manifest.yaml`. A conforming installer generates
that Toolkit-managed file from the installation contract with Toolkit,
Framework, AGENTS contract, installer, skill and capability facts plus install
time and a trustworthy source commit or null.

Project validation checks both manifests, their supported schema versions, the
installed-manifest path and installed skill agreement. Dynamic Git SHA, dirty
state and build time are generated by `Toolkit/scripts/New-SDPBuildIdentity.py`;
they are not editable project facts.
