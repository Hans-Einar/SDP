## High finding H1-R3 — Extended-path normalization can target the wrong project

`Toolkit/scripts/Install-SDP.ps1` removes a supported Windows `\\?\` drive or
UNC prefix, then calls `Path.GetFullPath` without proving that the raw extended
path and resulting ordinary path identify the same directory. NTFS can represent
distinct extended-only names whose segments end in a space or dot.

The Reviewer created distinct ordinary `project` and extended-only `project `
directories. Passing the extended-only directory as `ProjectRoot` produced an
applicable 25-action plan, and apply populated the ordinary `project` directory
while leaving the explicitly supplied extended-only directory untouched. This
is wrong-project mutation and contradicts the documented fail-closed boundary.

Required remediation: reject every normalization-sensitive extended drive/UNC
input before prefix removal, or prove raw and normalized OS identities equal
before substitution. Add trailing-space and trailing-dot regressions for project
and backup/source boundaries proving PlanJson and apply fail with zero mutation.
Canonical extended drive/UNC inputs must remain supported.
