## Residual findings

### H1-R2 — Physical root aliases bypass source/target overlap

`Toolkit/scripts/Install-SDP.ps1` compares source, project and backup roots by
lexical path. The Reviewer reproduced a Windows local-drive path and UNC path
that identify the same physical SDP directory but compare differently and have
no reparse point. `-PlanJson` returned `canApply: true` with 25 actions when the
aliased source directory was used as the project root.

Required remediation: compare stable physical/device identity across local,
UNC, extended and available short-name aliases for source/project and
backup/source overlap before planning; add local/UNC and backup/source
regressions that make zero mutations.

### H3-M8-R2 — Destination topology can fail after partial mutation

Preflight checks exact destination keys but not ancestor/descendant topology.
The Reviewer reproduced both:

- a schema-valid manifest with `AGENTS.md` and `AGENTS.md/README.md`
  destinations; and
- the canonical manifest with an existing target file named `.codex`, which is
  an ancestor of managed skill destinations.

Both produced applicable plans. Apply wrote earlier files and then failed at the
conflicting child, leaving a partially mutated target.

Required remediation: reject every manifest destination prefix conflict and
every existing non-directory destination ancestor before an applicable plan is
emitted. Add mutated-contract and canonical-target regressions proving plan
blocking and zero mutation.

### M10-R2 — Python enforces only three of seven governing pairs

PowerShell enforces all seven documented schema/capability pairs, but
`Toolkit/scripts/validate_sdp.py` covers only CurrentIndex, Relations and Ledger.
The Reviewer rebound the Fix-record, Release-record, project-manifest and
installed-manifest schemas to another declared capability and Python contract
validation returned no errors.

Required remediation: define all seven canonical pairs once in Python and add a
parameterized regression for every governed schema, preserving agreement with
PowerShell and the public pairing table.
