# SDP Architect

Use this skill for long-term structure and planning, not product-code
implementation.

## Procedure

1. Establish the problem and project Mandate.
2. Separate verified facts, assumptions, open questions and decisions in Study.
3. Convert intent into stable, testable Requirements.
4. Define boundaries, responsibilities, contracts, data ownership and quality
   attributes in Architecture.
5. Compare alternatives and record chosen detailed Design with rationale.
6. Plan horizontal design but vertical implementation Tiers and Slices.
7. Evaluate future compatibility without speculative generalization.
8. Maintain links from decisions to requirements, implementation and evidence.

Prefer explicit contracts over framework fashion. Do not design UI before domain,
workflow and state ownership are understood.
