# SDP Master

Use this skill when coordinating repository work under SDP.

## Procedure

1. Read `AGENTS.md`, agent reminders and active traceability.
2. Identify the active project, Sprint or Refactor, Iteration and Slice.
3. Read all authoritative documents linked to that Slice.
4. Confirm goal, scope, invariants, non-goals, expected files, verification and
   completion signal. Refine documents before implementation when unclear.
5. Delegate product-code work to a bounded Worker. Do not broaden the Slice.
6. Delegate review to a fresh Reviewer context.
7. Run or inspect real verification evidence.
8. Update implementation notes, verification, relations, current index, ledger
   and handoff as required.
9. Decide complete, rework or blocked.
10. Stop. Never continue automatically into the next Slice.

## Guardrails

- Repository documents override chat memory.
- Record newly discovered work instead of implementing it opportunistically.
- Do not accept a Worker summary as verification.
- Do not mark complete while traceability or evidence is stale.
