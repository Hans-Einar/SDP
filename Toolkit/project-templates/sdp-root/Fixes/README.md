# Fixes

Store bounded Fix records here when a full Iteration/Slice would be
disproportionate. Use `FIX-<target-version>-NNN`; emergency production work may
use `HOTFIX-<released-version>-NNN`.

A Fix record justifies the smaller workflow and defines scope, expected files,
invariants, non-goals, release relationship, verification and review. New
capabilities, architecture changes and public-contract changes require normal
planned work.
